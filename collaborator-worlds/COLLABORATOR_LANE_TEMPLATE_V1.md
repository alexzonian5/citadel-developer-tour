# COLLABORATOR_LANE_TEMPLATE_V1

## Status
Active implementation template.

## Purpose
Turn the Ari lane into the first reusable collaborator-lane class so future trusted collaborators can receive bounded serious local surfaces without reinventing the architecture each time.

---

## Core rule
A collaborator lane is:
- a separate local surface
- under shared constitutional law
- bounded by explicit artifact and memory permissions
- equipped for real usefulness
- protected by rollback, logging, and escalation

It is not:
- Alex-core access
- a toy demo
- unrestricted inheritance
- a broad social-delegation surface by default

---

## Standard folder structure
Recommended root:
- `projects/Citadel/collaborators/<name>/`

Within it:
- `allowed/`
- `logs/`
- `meta/`
- `session/`

---

## Standard required files
### meta/
- `allowed_artifacts_manifest.md`
- `authority_and_boundary.md`
- `logging_policy.md`
- `rollback.md`

### session/
- `<NAME>_SURFACE_BEHAVIOR_V1.md`
- `<NAME>_FIRST_SESSION_SIMULATION_V1.md`

### allowed/
- allowed-surface schema
- first-session design
- access/elevation protocol if individualized
- approved project/task artifacts only

### logs/
- `session_log.md`

---

## Required design decisions per collaborator
1. What is this collaborator's allowed level?
2. What lane is actually useful for them?
3. What should they never see?
4. What should escalate to Alex?
5. What first-session win will make the lane feel serious and empowering?

---

## Standard safeguards
- no direct access to Alex-core memory
- explicit artifact whitelist
- no unrelated relationship notes
- no cross-front browsing by default
- clear authority note
- logging and rollback present before use

---

## Promotion rule
A collaborator lane may become richer only after:
- repeated usefulness
- stable boundaries
- no discomfort or confusion
- no leakage
- no authorship drift
- proof that the current level is working

---

## Friction rule
If collaborator-lane creation starts feeling easy enough to mass-produce, slow down.
Each lane is a trust surface, not just a folder template.
Scale should follow proof and stewardship, not enthusiasm.

---

## Final judgment
The collaborator lane should become a reusable class.
But every instance still deserves real design, explicit boundaries, and constitutional seriousness.
