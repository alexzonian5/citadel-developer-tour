# Live Command Board (Sanitized Public Snapshot)

## Canonical live-surface idea
Citadel distinguishes between:
- constitutional law
- current state
- active command surface
- archive/support material

Only a small subset of files should steer live action.
Everything else is reference, reserve, or archive unless re-promoted.

## Mission focus
Make the machine metabolically healthier by:
- preferring retrieval before regeneration
- preferring local-first routine routing
- tightening closure and writeback
- reducing stale influence in operator steering
- keeping the control plane understandable

## Immediate operating rule
Do not add new substrate power until the coordination node is governed.

## Public-safe readiness gates
Before widening the system:
1. expensive-model leakage is identified or stopped
2. scheduled loops are classified
3. state surfaces obey active-mission law
4. one reliable health-check method exists for each core organ
5. node contracts are accepted before heavy compute is added

## Operating philosophy
- live truth beats stale doctrine
- routing beats ad hoc handling
- receipts beat vague completion claims
- bounded membranes beat undifferentiated access
- workflows stay live only if they still produce artifacts or useful state change
