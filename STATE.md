# Citadel State (Sanitized Public Snapshot)

> This is a redacted architecture-facing state snapshot, not the raw private operator state.

## Active mission law
Citadel preserves a maximum-three-active-missions rule.

The public shape is:
- one or more revenue-near mission fronts
- one infrastructure / substrate mission
- explicit blockers and next actions per mission

## Current architectural emphasis
The system was recently tightened around these priorities:
- reduce uncontrolled background autonomy
- clamp premium-model drift
- keep the control plane legible
- enforce live truth over stale narrative surfaces
- prepare for heavier inference only after governance is clean

## Control-plane law
Keep the persistent coordination spine lean.
Typical keep set includes:
- state and truth surfaces
- retrieval / memory infrastructure
- gateway / orchestration runtime
- bounded backup and receipt flows
- only the workflow organs that still prove artifact throughput

## Cleanup law
Before adding new system power:
1. classify active loops as keep, freeze, archive, or unknown
2. remove stale influence from live control surfaces
3. ensure every live workflow has a health check and disable path
4. ensure expensive cognition is explicitly gated

## Premium-containment law
Publicly shareable rule:
- no autonomous premium recursion
- no silent premium failover loops
- local-first or deterministic-first by default
- expensive cognition only when justified

## Node expansion law
A heavier compute node may join only when:
- the control plane is governed
- routing law is explicit
- receipts and return contracts are defined
- the heavy node is labor, not authority
